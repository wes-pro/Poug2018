about challenge

Consolidate hundrends of databases on Oracle SuperCluster systems¶
looking for possible solutions

Ideal solution should be able¶

To have a overall view on platform performance¶

To gather information about ALL running instances¶

To keep track of platform resources while progressing with migrations¶

To accumulate and visualize performance data from many databases together¶

To understand topology of the platform (e.g. shared cpu pools)¶

cloud control possible solutions

Obvious answer: Oracle Enterprise Manager¶

Pros¶

Comprehensive source of performance data

Capability (limited) to consolidate data from multiple targets (Dynamic Groups or Problem Analysis)

Live and historical data is available

Cons¶

Not possible to create custom charts without BI Publisher

Doesn't automatically take hardware topology to create dynamic groups

Somebody needs to manually add database to monitoring

bi publisher possible solutions

Pros¶

our solution
- case #2

Wrong results and impact of *cell passthrough* workaround

Bug in Oracle code caused wrong results for some queries (bug in interval function)¶

1st temporary workaround: _kcfis_cell_passthru_enabled=true¶

2nd temporary workaound: cell_offload_parameters="OPT_DISABLED={283,284}";¶
